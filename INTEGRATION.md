# Unify — بوابة الطالب الذكية (تحديث كبير)

هذا التحديث حلّل نسخة الباك-إند الجديدة (Unify.rar) وأضاف **9 وحدات كاملة جديدة** للواجهة،
بالإضافة لتعديلات على تصميم الشاشات حسب الطلب.

## 1. أهم التغييرات على الواجهة

- **حذف "نسيت كلمة المرور" و OTP بالكامل من الويب** (البادج بالباك-إند ما زالت موجودة، بس ما في
  شاشة لها بالواجهة).
- **الشريط الجانبي (Sidebar) أصبح على اليسار وبالإنجليزية بالكامل**، بينما باقي الواجهة (العناوين،
  الجداول، النماذج، الإشعارات) بقيت عربية بالكامل.
- **حذف كل الإيموجي** من الواجهة (رسالة الترحيب، رسائل التنبيه...).
- **تبسيط شاشة "حسابي"**: حذف كل النصوص التوضيحية، بقيت البيانات فقط.

## 2. لا يوجد نظام أدوار/صلاحيات بالباك-إند حالياً

بحثت بكل ملفات الباك-إند (`Modules/**`) عن أي إشارة لـ `role` أو صلاحيات — **لا توجد أي نتيجة**.
يعني:
- **كل من يسجّل دخول بحساب صالح، يرى نفس لوحة التحكم الكاملة** (لا فرق بين Admin وLecturer
  بالوصول الفعلي، رغم أن Use Case Diagram يوضّح دورين منفصلين).
- مخطط الـ Use Case اللي أرسلته يمثّل **التصميم المستهدف مستقبلاً**، لكن الفصل الفعلي بين
  الأدوار (`role` بجدول users أو middleware صلاحيات) لسا غير مُنفّذ بالباك-إند.
- إذا حابب نفصل الأدوار فعلياً، لازم إضافة عمود `role` (أو ربط بجدول Spatie permissions) بالباك-إند
  أولاً، وبعدها نضيف Route Guards بالواجهة حسب الدور.

## 3. الوحدات الجديدة المُضافة للواجهة (من الباك-إند الجديد فقط)

| الوحدة | الصفحة | نوع العمليات |
|---|---|---|
| Courses | `/dashboard/courses` | CRUD كامل — مع متطلبات سابقة وربط بمناهج الأقسام (سنة/فصل دراسي) |
| Course Sections | `/dashboard/course-sections` | CRUD كامل — الباك-إند يتحقق أن شعبة "نظري" تُسند لدكتور، و"عملي/مشروع" لمعيد مخبري |
| Classrooms | `/dashboard/classrooms` | CRUD كامل |
| Semesters | `/dashboard/semesters` | CRUD كامل — فصل واحد فقط يكون "حالي" بأي وقت |
| Schedules (SectionSchedule) | `/dashboard/schedules` | CRUD كامل — الباك-إند يمنع تعارض القاعة/المدرّس/الشعبة تلقائياً |
| Teaching Staff | `/dashboard/teaching-staff` | CRUD كامل — كل عضو مرتبط بحساب مستخدم + نوع (دكتور/معيد) + أقسام |
| Payments | `/dashboard/payments` | **عرض + تسجيل فقط** — الكونترولر بالباك-إند لا يملك update/destroy بعد |
| System Settings | `/dashboard/settings` | إعداد واحد (Singleton) لسعر الساعة المعتمدة — عرض/إنشاء/تعديل |
| Users | `/dashboard/users` | **عرض فقط (read-only)** — دوال store/update/destroy معلّقة (commented) بـ `UserController` |

الوحدات الموجودة سابقاً (Students, Departments) بقيت كما هي مع تحديث بسيط:
- الباك-إند الآن يفرض `phone` كحقل **مطلوب** عند إنشاء طالب (كان اختيارياً بالنسخة السابقة).
- موديول Departments أصبح يستخدم `ApiResponseTrait` (استجابة `{success,data}` بدل الخام) — تعامل
  الواجهة مع هذا تلقائياً لأنه `apiClient.js` يدعم الشكلين أصلاً.

## 4. ملاحظات تقنية مهمة اكتشفتها بالباك-إند الجديد

- **Payments**: `GET /api/payments` يرجع مصفوفة كاملة بدون Pagination.
- **System Settings**: الراوت `apiResource` مسجّل بالكامل لكن الكونترولر ما فيه `index()` ولا
  `destroy()`، و`show()`/`update()` بالكونترولر بلا معامل `id` (يتجاهلان أي id بالمسار). لهيك
  الواجهة تستخدم مساراً ثابتاً `/systemsettings/1` دائماً (القيمة الرقمية بحد ذاتها متجاهلة
  بالباك-إند).
- **Course Sections**: عند اختيار "نظري" بالنموذج، قائمة المدرّسين تُفلتر تلقائياً لتظهر فقط من
  نوعهم `doctor`؛ وعند اختيار "عملي/مشروع" تُفلتر لـ `lab_instructor` — مطابقة لقاعدة العمل
  بالباك-إند (`CourseSectionService`).
- **يوجد موديول مكرّر بالباك-إند** باسم `TechingStaff` (بدون حرف a — خطأ إملائي) وهو **فارغ تماماً**
  (بدون migration ولا كود فعلي) — الوحدة الصحيحة المُستخدمة هي `TeachingStaff`. لم تُربط الواجهة
  بالموديول الفارغ.
- لا توجد بالباك-إند حالياً أي وحدات لـ: Complaints، Notifications، Attendance/NFC Verification،
  Projects، Grades، Lectures — رغم ظهورها بمخطط الـ Use Case. هذي ميزات مستقبلية لسا ما انبنت
  بالباك-إند، فما قدرت أربطها.

## 5. التشغيل

```bash
cd smart-portal-login-react
npm install
npm run dev
```
`.env` يبقى كما هو:
```
VITE_API_BASE_URL=http://127.0.0.1:8000
```
